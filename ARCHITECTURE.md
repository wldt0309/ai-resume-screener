## 2. 技术选型

| 组件 | 技术栈 | 选型理由 |
|------|--------|---------|
| Web UI | Streamlit | 快速原型，数据可视化强 |
| 简历解析 | pdfplumber + python-docx | Python 原生，兼容性好 |
| JD 解析 | Rule-based + LLM | 无 LLM 时规则兜底，有 LLM 时增强 |
| 评分引擎 | 自研 | 胜任力模型定制化 |
| 数据库 | SQLite | 零配置，单文件，适合本地场景 |
| LLM | OpenAI/Anthropic/阿里云/MiniMax | 按环境自动检测 |

## 4. 关键技术决策

### 决策 1：规则 + LLM 混合解析
- **问题**：纯规则解析在复杂简历上准确率有限
- **方案**：先用规则快速解析，LLM 作为增强（可选）
- **收益**：无 API Key 时可正常运行，有 API Key 时准确率更高

### 决策 2：SQLite 单文件数据库
- **问题**：需要持久化但不想运维数据库服务
- **方案**：SQLite，数据存在 `data/resume_screener.db`
- **收益**：零配置，文件可拷贝备份

### 决策 3：胜任力评分模型
- **问题**：如何量化简历与 JD 的匹配度
- **方案**：4 维度加权（技能 40% + 经验 30% + 学历 15% + 语义 15%）
- **收益**：评分透明，可解释
